# ExampleSubject_Data

This folder represents the **Subject_Data** side of a governed project.

In Synapse terms, Subject_Data holds continuity and governance-facing artifacts, such as:
- codex and design notes
- snapshots and decision records
- quest boards and execution receipts
- guild orders or scoped work directives
- project history and rationale

This example folder contains tiny illustrative artifacts. They are not canonical law.
