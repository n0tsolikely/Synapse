================================================================================
0) PURPOSE
================================================================================
Control Sync is alignment before action.

This checklist is the deterministic ignition key for every session/day:
- prevents missing-state errors
- prevents intent-reality inversion
- ensures correct gating (Fog of War vs Fog Lifted)
- ensures audits + snapshots are anchored correctly

This checklist ADDS rigor.
It does not replace the Control Sync law document:
- Processes/SYNAPSE_GUILD__CONTROL_SYNC.txt

================================================================================
1) CHECKLIST (RUN IN ORDER)
================================================================================

1.1 Canonical setup confirmation (Truth Gate)
[ ] Governance pack extracted and treated as canonical (no edits in zips)
[ ] Subject pack(s) extracted and treated as canonical (if provided)
[ ] Canonical working tree declared (paths)

If any canonical directory is missing:
- STOP
- Disclosure Gate must notify Hands

----------------------------------------
1.2 Identify mode: Continuing vs New Subject
----------------------------------------
[ ] Continuing Subject (Subject_Data exists + Codex exists or prior work exists)
OR
[ ] New Subject (no Codex; subject being created from nothing)

If New Subject:
- run: Processes/SYNAPSE_GUILD__SUBJECT_INITIALIZATION_AND_INCUBATION.txt
- remain in Fog of War until Codex Freeze

----------------------------------------
1.3 Buff up (subject-local)
----------------------------------------
If <Subject>_Data exists:
[ ] Read <Subject>_Data/Buffs/<SUBJECT>_EXECUTION_PROTOCOL.txt
[ ] Read <Subject>_Data/Buffs/<SUBJECT>_DATA_DIRECTORY_MAP.txt
[ ] Read <Subject>_Data/Buffs/<SUBJECT>_SESSION_START_CHECK.txt

If continuity artifacts exist:
[ ] Read Continuity Lock (latest)
[ ] Read Bootstrap Prompt (latest)

----------------------------------------
1.4 Start-of-day snapshot review (MANDATORY)
----------------------------------------
Review order is locked:
[ ] Latest Control Sync Snapshot (intent/decisions)
[ ] Latest End-of-Day Snapshot (execution reality/outcomes/blockers)
[ ] Latest relevant General Snapshots (if any)

If any are missing:
- record that fact in today’s Control Sync Snapshot

----------------------------------------
1.5 World State gate (Fog of War vs Fog Lifted)
----------------------------------------
[ ] World State identified:
    - Fog of War (Incubation / Codex writing only; execution forbidden)
    - Fog Lifted (execution permitted)

If executing (Fog Lifted):
[ ] Codex Freeze marker exists:
    <Subject>_Data/Codex/CODEX_FREEZE.md

If marker is missing:
- STOP execution
- remain in Fog of War
- Disclosure Gate notify Hands

----------------------------------------
1.6 Review state surfaces
----------------------------------------
[ ] Active Guild Orders/Raids reviewed
[ ] Paused Guild Orders/Raids reviewed
[ ] Quest Board reviewed (BOARD + ACCEPTED + COMPLETED + ABANDONED)

----------------------------------------
1.7 Decide scope (Hands authority)
----------------------------------------
[ ] Decide today/session scope:
    - which quests will be accepted
    - which raids/dungeons are in focus
    - any explicit deferrals (and why)

----------------------------------------
1.8 Quest acceptance validation gate
----------------------------------------
For each quest that will be accepted:
[ ] Validate quest file format (see: Quest Board/SYNAPSE_GUILD__QUEST_VALIDATION_RULES.txt)
[ ] Create audit bundle folder and record its path in the quest file
[ ] Only then move quest to Accepted/

----------------------------------------
1.9 Write Control Sync Snapshot (mandatory)
----------------------------------------
[ ] Record:
    - snapshot review results
    - decisions made
    - scope accepted for this session
    - any notes that must carry forward
    - doc references if Hands requested “save note to docs”
[ ] Store it in:
    <Subject>_Data/Snapshots/Control Sync/

----------------------------------------
1.10 End Control Sync explicitly
----------------------------------------
Control Sync continues until Hands ends it.
[ ] Hands ended Control Sync
→ only then may Dailies execution begin.

END OF CHECKLIST
