SYNAPSE GUILD — QUEST VALIDATION RULES
Version: v2.0
Last Updated: 2026-03-31

================================================================================
1) CORE LAW
================================================================================
- A Quest is a bounded coherent outcome.
- Milestones / work items may be atomic; Quests are not required to be atomic.
- A Quest must be independently closable with one honest completion decision.
- If a plan contains multiple independently closable outcomes, it must split into multiple linked quests.

================================================================================
2) REQUIRED QUEST FIELDS BEFORE ACCEPTANCE
================================================================================
Every quest moving from BOARD to ACCEPTED must have:
- Quest ID / Title / Subject / Origin / Priority
- Codex Anchors (DRAFT)
- Codex Constraint Summary (DRAFT)
- Change Class / Vision Delta / System Context Statement
- Anti-Duplication Plan / Placement Intent
- Coherent Outcome
- Closure Statement
- Split Triggers
- Scope / Objective
- Stretch Plan / Milestones
- Out of Scope
- Dependencies (or explicit None)
- Door Impact + Testing Level when the quest touches code/software
- Concrete Verification Plan
- Plan Artifact Refs
- Audit Bundle Folder Path
- Talent Point Awarded (YES / NO)

================================================================================
3) VERIFICATION PLAN LAW
================================================================================
- Verification must be concrete before acceptance.
- “Deferred to 01_PREQUEST.md”, “deferred indefinitely”, or vague placeholders are invalid.
- The plan must state expected checks and what PASS / FAIL / BLOCKED mean.

================================================================================
4) RISK / CONSENT LAW
================================================================================
- Risk defaults to R0 if omitted.
- If Risk = R2, the confirmation artifact path must be present and the file must contain CONFIRM: YES.

================================================================================
5) WORLD-STATE / CODEx FREEZE GATE
================================================================================
- If Fog of War is active, acceptance is illegal.
- Codex Freeze marker must exist before governed execution begins.

================================================================================
6) COMPLETION LAW
================================================================================
A quest may move from ACCEPTED to COMPLETED only if:
- the audit bundle exists
- the latest 01_COMPLETION_AUDIT.md exists and parses
- milestone status is fully satisfied for in-scope work
- required verification ran and receipts exist
- no failed / blocked checks remain unresolved
- no skipped in-scope work, unresolved gaps, or known regressions remain unaccounted for
- Overall Verdict = PASS
- Final State Decision = COMPLETED

If any of those are false:
- the quest remains ACCEPTED (or reopens back to ACCEPTED)
- a new completion audit attempt is required after remediation

================================================================================
7) LEGACY COMPATIBILITY
================================================================================
- Historical 01_PREQUEST / 02_EXECUTION / 03_VERIFY / 04_OUTCOME bundles remain readable.
- Do not rewrite historical bundles unless the quest is actively re-completed.
- New writes must use 00_SUMMARY.md + 01_COMPLETION_AUDIT.md + append-only completion-audit history.

================================================================================
8) BUNDLE MINIMUMS FOR NEW WRITES
================================================================================
Required files:
- 00_SUMMARY.md
- 01_COMPLETION_AUDIT.md
- 06_CHANGED_FILES.txt
- 06_TESTS.txt
- 06_WRAPPER_PROOF.json
- 00_ACCEPTANCE_RECEIPT.txt
- 00_GOVERNANCE_PREFLIGHT.md
- DISCLOSURE_GATE.md
- 90_ORIGINAL_QUEST__as_found.txt

Optional:
- 05_DISCLOSURE.md
- 01_COMPLETION_AUDIT__ATTEMPT-###.md

================================================================================
9) WRAPPER / RECEIPT LAW
================================================================================
- Governed quest commands must run through synapse_quest_run.sh.
- Completion must flow through complete-quest.
- No proofless “done” claim is legal.

END OF DOCUMENT
