================================================================================
0) PURPOSE OF THIS DOCUMENT
================================================================================

This document defines what a BOOTSTRAP PROMPT is, why it exists, and how it is used.

After reading this document, the phrase “Provide a Bootstrap Prompt” MUST be sufficient
instruction for any compliant operator (human or AI).

A Bootstrap Prompt exists to solve one problem:

> Once all artifacts are loaded, how does a brand-new AI session know
> how to behave, what role it is in, and what to do next?

A Continuity Lock defines WHAT IS TRUE.
A Bootstrap Prompt defines HOW TO PROCEED.

This is governance law.

================================================================================
1) CORE DEFINITION (CANON)
================================================================================

1.1 Bootstrap Prompt — definition
A Bootstrap Prompt is a behavioral initialization artifact that instructs a new AI
session how to interpret, prioritize, and act upon the loaded artifacts at the start
of a session.

A Bootstrap Prompt establishes:
- role
- stance
- constraints
- allowed actions
- starting behavior

One-sentence definition:
A Bootstrap Prompt activates a new session by instructing the AI how to behave and
what to do now that all relevant artifacts are present.

================================================================================
2) WHAT PROBLEM IT SOLVES (WHY IT EXISTS)
================================================================================

Bootstrap Prompts exist because:

- AI sessions start with no memory of prior chats.
- Artifacts alone do not imply behavior.
- Different phases require different stances (design vs execution vs governance).
- Without guidance, AI will:
  - ask unnecessary questions
  - redesign settled decisions
  - propose scope when it should execute
  - execute when it should pause

A Bootstrap Prompt prevents behavioral drift at session start.

================================================================================
3) WHAT A BOOTSTRAP PROMPT IS — AND IS NOT
================================================================================

3.1 What it IS
A Bootstrap Prompt is:
- a role initializer
- a stance setter
- a behavior constraint
- a start instruction

3.2 What it is NOT
A Bootstrap Prompt is NOT:
- a state record
- a decision log
- a summary
- a Continuity Lock
- an Execution Pack

It does not define what is true.
It defines how to act given what is true.

================================================================================
4) VALIDITY RULES (MINIMUM REQUIRED CONTENT)
================================================================================

A Bootstrap Prompt may be written in any style, but it MUST contain the following
minimum content.

If any required item below is missing or ambiguous:
- the Bootstrap Prompt is INVALID
- rehydration is INCOMPLETE
- execution authority MUST NOT advance
- Disclosure Gate MUST trigger

4.1 REQUIRED — Role + stance
The Bootstrap Prompt MUST explicitly declare:
- who the AI is (role)
- what posture it is in (stance)

Examples (illustrative only):
- “You are Brains operating as Synapse. Conversation is input; files are authority.”
- “You are in Governance mode until Hands closes Control Sync.”

4.2 REQUIRED — Authority order (anti-drift)
The Bootstrap Prompt MUST state the authority order it will obey.
At minimum it MUST explicitly include:
- Locks override all other text.
- Filesystem artifacts are authoritative.
- Conversation is not an artifact.
- Inference is never used to advance state.

4.3 REQUIRED — What is allowed vs forbidden NOW
The Bootstrap Prompt MUST explicitly constrain behavior by stating:
- what the AI MAY do in this session
- what the AI MUST NOT do in this session

At minimum, it MUST address:
- whether execution is permitted
- whether scope decisions are permitted (or require Control Sync)
- whether it must wait for Hands

4.4 REQUIRED — What MUST NOT be re-litigated
The Bootstrap Prompt MUST instruct the AI not to redesign settled decisions.
It MUST identify the artifact that defines binding state (Continuity Lock) and state
that it is binding.

4.5 REQUIRED — Read priority / activation order
The Bootstrap Prompt MUST specify the activation order for loaded artifacts.
At minimum it MUST include:
- Buffs first (if present)
- Continuity Lock
- Codex (and Freeze marker if present)
- Control Sync / Dailies routing (as appropriate)

4.6 REQUIRED — First action
The Bootstrap Prompt MUST specify exactly what the AI does first after rehydration.

If the Bootstrap Prompt does not contain an explicit first action:
- default behavior is WAIT (do not proceed)
- request Hands to provide/repair the Bootstrap Prompt

================================================================================
5) BUFFS INTEGRATION (SESSION STARTUP LAW)
================================================================================

If a project provides Buffs (Execution Protocol, Directory Map, Session Start Check),
the Bootstrap Prompt MUST instruct the AI to read them BEFORE acting on any other
artifact.

Buffs are session ignition.
They reduce rehydration overhead while preserving strict governance.

================================================================================
6) RELATIONSHIP TO CONTINUITY LOCKS
================================================================================

Bootstrap Prompts and Continuity Locks are complementary but distinct.

Continuity Lock:
- Declares state
- Declares binding decisions
- Declares resume point

Bootstrap Prompt:
- Declares behavior
- Declares stance
- Declares starting action

Rule:
A Continuity Lock answers “What is true?”
A Bootstrap Prompt answers “What do I do with this truth now?”

A Bootstrap Prompt MUST NOT contradict a Continuity Lock.
If a contradiction exists:
- Continuity Lock governs
- Disclosure Gate MUST trigger

================================================================================

If multiple Bootstrap Prompts exist:
- the one with the latest date token is authoritative
- older ones MUST be archived

================================================================================
8) WHEN A BOOTSTRAP PROMPT IS REQUIRED
================================================================================

A Bootstrap Prompt is REQUIRED for rehydration of a continuing Subject.

If `<Subject>_Data/` exists and a session is resuming work:
- a Bootstrap Prompt MUST exist on disk
- if it does not exist: the Rehydration Pack is INCOMPLETE
  (see `Continuity/SYNAPSE_GUILD__REHYDRATION_PACK_CHECKLIST.txt`)

A Bootstrap Prompt MAY be omitted only if:
- Hands explicitly declares: “Exploration only; no state changes; no execution.”
- AND the session will not create, modify, move, or complete any artifacts

In that exception case:
- the AI MUST operate in Fog of War posture
- the AI MUST NOT advance state
- the AI MUST NOT create binding decisions

================================================================================
9) HOW A BOOTSTRAP PROMPT IS USED IN A NEW SESSION
================================================================================

Typical rehydration flow (continuing Subject):

1) Start a new AI session
2) Load all required artifacts (subject-state + governance pack)
3) Validate the Rehydration Pack
   → `Continuity/SYNAPSE_GUILD__REHYDRATION_PACK_CHECKLIST.txt`
4) Process the Bootstrap Prompt (behavior activation)
5) Perform the declared first action

Buffs ignition rule:
- If Buffs exist, read Buffs first:
  - `<SUBJECT>_EXECUTION_PROTOCOL.txt`
  - `<SUBJECT>_DATA_DIRECTORY_MAP.txt`
  - `<SUBJECT>_SESSION_START_CHECK.txt`

Rule:
The Bootstrap Prompt is processed AFTER artifacts are available.
It assumes artifacts exist; it does not replace them.

================================================================================
10) FORMAT AND STRUCTURE (FLEXIBLE FORMAT, FIXED CONTENT)
================================================================================

A Bootstrap Prompt is usually a single block of text.

Format is flexible.
Minimum content (Section 4) is NOT flexible.

Recommended (non-binding) structure:
- Role / stance
- Authority order
- Constraints (allowed vs forbidden)
- “Do not re-litigate” rule + binding artifact
- Activation order (Buffs → Lock → Codex)
- First action

================================================================================
11) CANONICAL COMMANDS
================================================================================

Once this law is accepted, the following phrases are valid:

- “Provide a Bootstrap Prompt.”
Meaning:
Provide a valid Bootstrap Prompt artifact that satisfies Section 4.

- “Rehydrate using this Bootstrap Prompt.”
Meaning:
Adopt the specified role, stance, constraints, and starting action.

- “Update the Bootstrap Prompt.”
Meaning:
Behavior or phase has changed; replace the prior prompt and archive the old one.

================================================================================
12) QUALITY BAR (WHAT MAKES A GOOD BOOTSTRAP PROMPT)
================================================================================

A good Bootstrap Prompt:
- prevents re-litigation
- prevents early drift
- starts work immediately or correctly pauses
- makes the first response predictable
- makes the legality of execution explicit

A bad Bootstrap Prompt:
- is vague
- asks the AI to infer intent
- mixes decisions and behavior
- causes the AI to ask unnecessary questions
