TALENT LOG — SYNAPSE GUILD (append-only)
Version: v1.1
Last Updated: 2026-02-08

Purpose
- Progression history.
- Records when Talent Points were earned and what they were spent on.

Canonical placement
- This file is a TEMPLATE inside the Governance pack.
- The ACTIVE Talent Log for a Subject lives at:
  <Subject>_Data/Talent Tree/TALENT_LOG.txt

Hard rules
1) Append-only
   - Never delete entries.
   - Never rewrite history.
   - Corrections are appended as new entries that explicitly reference the prior entry being corrected.

2) Artifact reality
   - If an entry is not written to the filesystem, it does not exist.
   - Chat summaries do not count as log entries.

3) Mode Awareness / Truth Gate compliance
   - If the current execution surface cannot write files, you MUST NOT claim the Talent Log was updated.
   - Instead, you MUST emit a PROPOSED TALENT LOG ENTRY (clearly labeled PROPOSED) and include the exact path where it must be written.

4) Deterministic linkage
   - Every Talent Point award MUST be traceable to a specific Quest and its completion artifacts.
   - Minimum required references:
     - Quest ID + canonical quest file path
     - Verification/Outcome evidence path(s)

Entry format (one award = one entry)
- Timestamp (timezone required)
- Quest reference
- Award decision (YES/NO)
- Spend decision (Talent ID + Name OR NONE)
- Evidence paths
- Notes (optional)
