# LOCK — Codex Freeze (Fog Lifted)
**Locked:** 2026-03-12 00:00:00 (America/Toronto assumed)

## Canon meaning
Codex Freeze is the ONLY gate that lifts World State from **Fog of War** to **execution-legal**.

If Codex Freeze cannot be proven, World State MUST be treated as Fog of War.

Codex Freeze does **not** mean the Codex has become a perfect, final, never-changing encyclopedia.
Codex Freeze means the Codex spine is stable enough that governed execution can proceed
without the Subject constantly redefining itself underneath the work.

In one sentence:
- **Freeze means stable enough for governed execution, not complete beyond all future change.**

## Gate Condition (Fog Lifted)
Fog is lifted ONLY when ALL of the following are true:

1) **Marker file exists on disk** at:
   - `<Subject>_Data/Codex/CODEX_FREEZE.md`

2) **Brains approval is proven by the marker content**.
   - Chat statements ("freeze it") are not sufficient.
   - The marker MUST contain:
     - a freeze timestamp (date/time)
     - an explicit Brains approval line (e.g., `Brains: <name>` or `Brains Approval: YES`)

3) **Codex Freeze is not contradicting known Codex reality**.
   If the current Codex structure, constraints, or governing meaning have changed since the marker was written
   and the marker was not updated accordingly, then Freeze is **STALE** and must be treated as **not proven**
   until corrected (Disclosure Gate).

If any condition above is missing, unreadable, or ambiguous:
- Fog of War remains active
- any attempt to accept Quests / start a Raid / execute governed work must trigger Disclosure Gate

## Effects (Once Closed)
When Codex Freeze is proven (Fog Lifted):
- Quests may be ACCEPTED for execution (BOARD → ACCEPTED) under Quest Validation Rules
- Guild Orders may be executed as a Raid (ACTIVE state)
- Governed execution is permitted under Dailies, with Truth Gate + Execution Audits enforced
- The system may truthfully claim `governed_execution_ready` when the remaining execution prerequisites are also satisfied

Codex Freeze does NOT override:
- Truth Gate
- Disclosure Gate
- Risk Rubric + Consent Gate
- Canonical Working Tree / Engine vs Data boundaries

## What is allowed BEFORE Freeze (Fog of War)
Allowed:
- Exploration / Incubation / Discoveries
- Drafting TOC and Codex sections
- Drafting Build Manual material
- Drafting Guild Orders in PAUSED
- Drafting Quests on BOARD as proposals / canonical board artifacts (not governed execution commitments)
- Ambient quest detection, candidate clustering, and lawful BOARD drafting
- Snapshots and continuity artifacts
- Ambient or exploratory work that remains non-governed and non-accepted

Forbidden until Freeze is proven:
- Accepting Quests (BOARD → ACCEPTED)
- Starting a Raid / executing Guild Orders as governed work
- Claiming governed execution readiness
- Claiming exploratory movement as already authorized governed history

## Controlled changes AFTER Freeze
Codex changes remain allowed, but they MUST be deliberate:
- changes occur during Control Sync
- the Control Sync Snapshot MUST record what changed and why

If a Codex change affects structure, constraints, or governing meaning:
- Brains must explicitly re-authorize Freeze
- `<Subject>_Data/Codex/CODEX_FREEZE.md` MUST be updated

If the marker cannot be updated or cannot be proven:
- treat World State as Fog of War until corrected

## Doctrine note — Codex maturity
The Codex may still evolve after Freeze.
What Freeze certifies is not perfection.
What Freeze certifies is that the Subject's spine is now stable enough for governed execution.

The Codex may therefore continue to gain:
- detail
- examples
- clarifications
- new lawful sections

without pretending that every post-Freeze improvement means the earlier Freeze was invalid.
Only changes that alter structure, constraints, or meaning require re-freeze.

## Do not assume
- Do not infer Freeze from "we have a Codex."
- Do not infer Freeze from an assistant claiming it.
- Do not treat partial drafts as execution readiness.
- Do not confuse Codex maturity with Talent Tree maturity or Build Manual completeness.
