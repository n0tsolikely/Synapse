# TERM — Canonical Working Tree (No Parallel States)

## Definition
The Canonical Working Tree is the single authoritative representation of the current state of a Subject’s Engine and Data.

## Purpose
Prevents parallel states, hidden forks, and contradictory truths.

## Do Not Assume
- Canonical does not forbid experimentation; it requires explicit branching and explicit promotion
- Canonical does not auto-merge changes

## Interactions
- Truth Gate enforces canonical state
- Audits record state transitions
- Snapshots reference canonical state at capture time
