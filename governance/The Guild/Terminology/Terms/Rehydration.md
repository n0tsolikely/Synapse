# TERM — Rehydration (Bootstrap / Locks / Packs)

## Definition
Rehydration is restoring shared understanding from artifacts (not memory) so work resumes deterministically.

## Purpose
Prevents context loss across sessions/days/operators.

## Components (common)
- Bootstrap prompt or summary
- Continuity locks (idea/state anchors)
- Rehydration packs (engine + data + active subject folders)

## Do Not Assume
- Rehydration is not execution
- Rehydration does not guarantee correctness
- Rehydration is a restore step, not a build step

## Interactions
- Uses snapshots, audits, codex/map, talents
- Often begins a Control Sync
