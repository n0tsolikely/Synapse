# TERM — End-of-Day Snapshot

## Definition
An End-of-Day Snapshot records the execution reality: what was implemented, tested, what failed, why, and what artifacts changed.

## Purpose
It preserves execution receipts so unfinished work is not repeated blindly.

## Do Not Assume
- It is not a planning document
- It does not redefine scope; it reports outcomes

## Interactions
- Reviewed after the prior Control Sync Snapshot
- Feeds discussion and decisions in the next Control Sync
